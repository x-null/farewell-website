Migration

After the GameSpy shutdown, the in-game server browser stopped working and we had to switch to using an alternative masterserver developed by xNULL in-house. Since we don’t own gamespy.com domain, this means that MoH binaries and the Reborn patch distributed over the years point to the master.x-null.net domain.

Masterserver is a system that makes the online server list work, game servers and game clients need to connect to it.

master.x-null.net domain will continue to work till 2027-03-01 to ensure that everything still works and to give everyone enough time to transition to other solutions.

You have several different options that you can choose from, depending on your preferences. What's important is that, no matter which option you choose now, you will have exactly the same experience when it comes to the masterserver. Same server list and everything. You are also not obliged to pick one option and stick with it forever. You can change your mind at any time.

You need to update your game before 2027-03-01. You have to do that!

Running on Linux

Remember to set files as executable with chmod +x <main executable>, for example chmod +x mohaa_lnxded.

On modern x64 bit distros, you need to install 32bit compatibility layer to run MoH servers. For a Debian-based distro that would be:

sudo dpkg --add-architecture i386
sudo apt-get update
sudo apt-get install libstdc++5:i386

Online Patching Tool

Visit MoH Reborn › Getting started › Manual Online Patching for more.

An easy to use patching tool. You drag & drop your binaries and the tool automatically detects if they need patching. You can patch your binaries with a click of a button and everything happens locally on your computer! There is no upload involved!

Use this option if you have all the required files and you only want to patch the game on your own to have working in-game server browser.

Keep in mind that with manual patching, you only make the in-game server browser work. You won't get any additional fixes or features that come with full Reborn installation.

333networks

Website: 333networks.com

333networks team, with the help of people that work on OpenMoHAA, prepared dedicated instructions on how to use their masterservers directly. You can find their instructions at https://333networks.com/instructions/mohaa .

The instructions are basically the same as the ones for the MoH Reborn with the exception that 333networks binaries point to master.gonespy.com masterserver domain, while MoH Reborn uses the master.gametir.com domain.

If you don't want to patch your binaries yourself, or you don't want to use MoH Reborn's masterserver domain, but instead you would prefer to make your game connect directly to one of the masterserver domains affiliated with 333networks, you can use their instructions and already patched binaries they provide.

OpenMoHAA

Website: openmohaa.org

OpenMoHAA uses 333networks masterserver by default.

If you use OpenMoHAA you don't have to do anything more. Your game will work just fine.

However, if you would like to use different masterserver with OpenMoHAA, for example master.gonespy.com (333networks affiliate) or master.gametir.com (provided by MoH Reborn) you need to change your server settings.

OpenMoHAA supports multiple masterservers all at once. By default, it uses following masterservers:

  • master.333networks.com
  • cvars:
    • com_master0_host: master.333networks.com
    • com_master0_msport: 28900
    • com_master0_hbport: 27900
  • master.frag-net.com
  • cvars:
    • com_master1_host: master.frag-net.com
    • com_master1_msport: 28900
    • com_master1_hbport: 27900
  • master.errorist.eu
  • cvars:
    • com_master2_host: master.errorist.eu
    • com_master2_msport: 28900
    • com_master2_hbport: 27900
  • master-au.unrealarchive.org
  • cvars:
    • com_master3_host: master-au.unrealarchive.org
    • com_master3_msport: 28900
    • com_master3_hbport: 27900
  • master.noccer.de
  • cvars:
    • com_master4_host: master.noccer.de
    • com_master4_msport: 28900
    • com_master4_hbport: 27900

To change one of these masterservers, all you need to do is to configure appropriate cvar in your server.cfg file (for server) or omconfig.cfg (for players), for example: 


seta com_master0_host "master.gonespy.com"
seta com_master0_msport "28900"
seta com_master0_hbport "27900"

To add new, additional masterserver, you can use cvar with incremented number, for example: 


seta com_master5_host "master.gametir.com"
seta com_master5_msport "28900"
seta com_master5_hbport "27900"

Warning

OpenMoHAA is in constant development!

At the moment, OpenMoHAA supports maximum of 8 masterservers, but this might change at any moment and this guide can quickly become out of date.

Make sure to check official OpenMoHAA documentation and ask for support on its GitHub project page or Discord server when something doesn't work.

Other

There are other places with MoH community resources.

If you use Volute, HaZardModding MoH mods or MoH Revival, you may need to download updates from each respective source.

Volute

Website: volute.io

Volute is a MoH:AA Anti-Cheat countering hacking and cheating in MoH:AA Multiplayer. Most community-driven organisations that run MoH leagues and tournaments require it in their official matches.

Brixton, the creator of Volute, updated MoH binaries provided with Volute to use master.gonespy.com masterserver.

Make sure to download the newest version and remember that if you want to use Volute, you have to use game binaries that are whitelisted and allowed by the anti-cheat.

  • Download MOHAA.exe from volute.io/downloads
